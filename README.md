# ai_test_tool

ai_test_tool 是一个基于 Python、DearPyGui 和 EvalScope 的桌面评测工具，用于集中管理大模型 API 配置，并执行图形化评测任务。当前版本已经支持模型配置管理、Needle-In-A-Haystack 捞针测试、后台任务执行、实时日志展示，以及评测结果的 HTML 报告与热力图查看。

## 项目定位

本工具面向需要对不同大模型接口进行本地化测试、对比与验证的个人开发者或研究者。当前主要用于长上下文检索能力验证，尤其适合快速执行 NIAH 捞针测试。

它可以帮助你：

- 维护多个模型 API 配置。
- 通过图形界面发起 NIAH 捞针测试。
- 在后台线程执行评测任务，避免界面阻塞。
- 查看运行日志、任务进度、热力图摘要和 EvalScope 生成的 HTML 报告。

## 当前已实现功能

### 模型配置管理

- 支持新增、编辑、删除模型配置。
- 每个配置包含配置名称、Base URL、API Key、Model Name。
- 配置会持久化到当前用户目录下的本地文件。
- 在 Windows 上，API Key 默认使用系统凭据保护能力进行加密存储。

### 捞针测试（NIAH）

- 支持选择待测模型配置。
- 支持单独选择裁判模型配置。
- 支持设置检索问题。
- 支持逐行填写多个 needles。
- 支持设置上下文长度范围与区间数。
- 支持设置文档深度百分比范围与区间数。
- 支持选择英文、中文语料子集。
- 支持设置 tokenizer 路径。
- 支持控制热力图是否显示数值。

### 结果与过程展示

- 底部日志面板会实时显示运行日志。
- 状态栏和进度条会反映后台任务状态。
- 运行过程中会读取 EvalScope 的 progress.json，显示已处理数量、百分比等进度信息。
- 任务完成后会展示平均准确率、样本数、输出目录、热力图信息和 HTML 报告路径。
- 可直接从界面打开 EvalScope 生成的 HTML 报告。

## 运行环境

- Python 3.12 及以上
- uv
- Windows 为当前主要使用环境

项目依赖已在 pyproject.toml 中声明，核心依赖包括：

- dearpygui
- evalscope[all]

## 安装与启动

安装依赖：

```bash
uv sync
```

启动程序：

```bash
uv run src/main.py
```

如果你在 PowerShell 中运行，也可以使用：

```powershell
uv run .\src\main.py
```

## 使用说明

### 第一步：配置模型

进入“模型配置”页面后，填写以下字段：

- 配置名称：用于区分不同模型配置。
- Base URL：模型服务接口根地址，例如云 API 地址或本地服务地址。
- API Key：访问模型接口所需密钥。
- Model Name：实际发送给服务端的模型名称。

填写完成后点击“保存配置”。

你可以：

- 在右侧列表中选择已有配置并回填到表单。
- 修改已有配置后再次保存。
- 选中列表项后删除配置。

建议至少保存两份配置：

- 一份作为待测模型。
- 一份作为裁判模型。

### 第二步：运行捞针测试

进入“测试任务”页面，完成以下设置：

1. 选择待测模型配置。
2. 选择裁判模型配置。
3. 输入检索问题。
4. 在 Needles 文本框中逐行填写待插入的事实。
5. 设置上下文长度最小值、最大值和区间数。
6. 设置文档深度最小值、最大值和区间数。
7. 设置 tokenizer 路径。
8. 选择语料子集（英文、中文）。
9. 按需勾选热力图显示数值。
10. 点击“运行捞针测试”。

### 第三步：查看结果

运行后，你可以关注以下区域：

- 底部日志面板：查看详细运行日志。
- 状态栏和进度条：查看任务整体状态。
- 右侧执行与展示区域：查看任务摘要、准确率、输出目录和报告入口。
- 热力图区域：查看不同上下文长度和插入深度下的命中情况。

如果 HTML 报告已经生成，按钮会变为可用状态，点击后会在浏览器中打开。

## 输出目录说明

EvalScope 运行结果默认会写入项目下的 outputs 目录。一次运行通常会产生如下内容：

- configs：任务配置快照。
- logs：执行日志。
- predictions：模型预测结果。
- reviews：裁判模型评审结果。
- reports：HTML 报告与结构化报告数据。
- progress.json：运行中进度信息。

## 当前限制

- 当前图形界面的“结果概览”页面仍为占位区域，尚未接入统一结果数据模型展示。
- 当前已完成并接入 GUI 的评测任务主要是 NIAH。
- 非 Windows 平台下，API Key 会退回为明文持久化策略，使用前请自行评估安全性。
- 裁判模型配置是运行 NIAH 的必填项，未选择时任务不会启动。

## 适用场景

- 验证不同模型在长上下文检索任务中的表现。
- 对接本地部署模型与云端模型接口，统一进行配置和测试。
- 快速查看 EvalScope 运行产物，而不必手动拼装任务配置。
